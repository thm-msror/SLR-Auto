You are helping to write a structured literature review summary.

You will receive a list of JSON objects, where each object corresponds to a screened paper and contains at least:
- title
- year
- key_technologies
- datasets
- task_type
- relevance (YES/NO/MAYBE)

⚠️ IMPORTANT RULES:
- Only use the information explicitly provided in the JSON.
- Do NOT invent papers, datasets, methods, or links.
- If a field is missing or unclear, write "N/A".
- Summaries must be based solely on the given input.

---

### Required Output Format (Markdown)

1. **Datasets**
   - Identify the top 10 most frequently used datasets.
   - For each dataset, mention the domain (if available in the JSON).
   - Report how many papers used it and the most common year.
   - Group similar datasets only if they appear together in the input.
   - Present results in a Markdown table.

2. **Methods**
   - Identify the 10 most common methods or technologies.
   - Show trends across years (*rising*, *stable*, *declining*), based ONLY on counts per year in the JSON.
   - Highlight any methods that appear only in the last 2 years (mark as *emerging*).
   - Present results in a Markdown table.

3. **Notable Papers**
   - Identify the most recent **relevant** paper (highest year, relevance=YES) and summarize it using only the fields available.
   - Identify the most recent paper overall and summarize it similarly.
   - If any paper is clearly labeled as a survey/review, summarize it. If not, say "No surveys found."

4. **Overall Trends**
   - Write a short narrative on how the field is evolving, based only on the datasets, methods, and years in the JSON.
   - Mention potential gaps or underexplored areas, but only if visible in the data.

Return your answer strictly as **Markdown text** with clear section headers, bullet points, and counts.
